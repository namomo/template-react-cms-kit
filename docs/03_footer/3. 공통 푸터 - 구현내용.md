# 03. 공통 푸터 (Footer) - 구현 내용

## 1. 개요
백오피스 서비스 성격이나 사용자 옵션에 따라 푸터(Footer) 영역의 노출 여부를 동적으로 전환할 수 있는 기능을 구현한 내역입니다.

## 2. 작업 내용
### 2-1. 공통 앱 설정 환경 분리
* **생성 파일**: `src/constants/app-config.js`
* 앱 전반에서 참조할 시스템 공통 설정 파일(`appConfig` 객체)을 생성했습니다.
* 레이아웃 제어 옵션인 `layout.showFooter` 속성(`true` / `false`)을 추가했습니다.

### 2-2. 공통 푸터 UI 컴포넌트 생성
* **생성 파일**: `src/components/layout/app-footer.jsx`
* `antd`의 `<Footer>` 컴포넌트를 사용해 중앙 정렬된 저작권 문구 등 기본적인 텍스트 요소를 표시하는 전용 컴포넌트를 분리했습니다. 
* 연도 표기 등을 `new Date().getFullYear()`로 동적으로 갱신되도록 처리했습니다.

### 2-3. 옵션 기반 레이아웃 렌더링 적용
* **수정 파일**: `src/layouts/protected-layout.jsx`
* 작성된 `AppFooter` 컴포넌트를 `ProtectedLayout` 하위, `<Content>` 태그 아래 위치시켰습니다.
* 무조건 렌더링되던 방식에서 벗어나 `{appConfig.layout.showFooter && <AppFooter />}` 처럼 옵션 설정 파일값에 의존한 **조건부 렌더링**을 적용하여 언제든지 레이아웃에서 끄고 켤 수 있게 되었습니다.

## 3. 완료 내역
* **기능 완료**: 시스템 설정 플래그 기반 Footer On/Off 기능 및 분리형 UI 구현.
