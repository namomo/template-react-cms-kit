# 03. 공통 푸터 (Footer) - 계획

## 1. 개요
`ProtectedLayout` 하단에 표시될 공통 푸터(Footer) 영역을 추가합니다. 카피라이트, 시스템 버전 등 기본적인 정보 표시용으로 사용되며, 비즈니스 요구사항에 따라 푸터 노출 여부를 동적으로 켜고 끌 수 있는 기능(옵션화)을 포함합니다.

## 2. 목적
*   **정보 제공**: 저작권(Copyright) 명시 및 어드민 시스템 버전 정보 등을 제공합니다.
*   **유연성**: 단순 화면 넓게 쓰기를 선호하거나 푸터 정보가 필요 없는 테넌트/사용자를 위해 글로벌 설정(`app-config.js`)으로 푸터를 On/Off 할 수 있도록 구성합니다.

## 3. 구현 내용 및 구조
### 3-1. 공통 설정 파일 분리 (`src/constants/app-config.js`)
*   앱 전체에 영향을 미치는 여러 설정값(예: 로고, 사이트 이름, 푸터 표시 여부 등)을 한 곳에서 중앙 관리하기 위해 `app-config.js`를 신규 생성합니다.
*   `showFooter: true | false` 속성을 정의합니다.

### 3-2. 푸터 컴포넌트 생성 (`src/components/layout/app-footer.jsx`)
*   Ant Design의 `<Footer>` 레이아웃 컴포넌트를 기반으로 텍스트 중앙 정렬 등 기본 스타일이 적용된 `AppFooter` 컴포넌트를 만듭니다.

### 3-3. Layout에 적용 (`src/layouts/protected-layout.jsx`)
*   `ProtectedLayout`에서 설정 값을 불러와 `appConfig.showFooter` 값이 `true`일 때만 `<AppFooter />`를 `<Content>` 아래 영역에 렌더링하도록 조건부 렌더링(Conditional Rendering)을 추가합니다.

## 4. 진행 순서
1. `src/constants/app-config.js` 파일 작성 및 설정 정의
2. `src/components/layout/app-footer.jsx` 푸터 UI 컴포넌트 생성
3. `src/layouts/protected-layout.jsx` 파일을 수정하여 구성 옵션에 따른 푸터 컴포넌트 추가
