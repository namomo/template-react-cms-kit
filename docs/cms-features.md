# CMS 기본 기능 구현 목록 제안

현재 `template-react-cms-kit`는 로그인과 라우팅 등 가장 핵심적인 뼈대만 갖추고 있습니다. 보편적이고 실용적인 어드민/CMS(Content Management System)로 발전하기 위해 필수적으로 추가되어야 할 기능과 UI 요소들을 다음과 같이 제안합니다.

## 1. 공통 UI & 레이아웃 프레임워크 강화
현재는 단순한 로그인 구조만 있으므로, 어드민 페이지다운 기본 화면 레이아웃이 필요합니다.

*   **사이드 탐색 바 (Sidebar/LNB)**
    *   메뉴 메뉴 관리 (접기/펴기 기능)
    *   현재 활성화된 메뉴 하이라이팅 기능
*   **상단 헤더 바 (Header/GNB)**
    *   현재 로그인한 사용자 프로필 및 알림(Notification) 아이콘
    *   로그아웃 버튼 및 계정 설정 팝업
    *   브레드크럼(Breadcrumb): 현재 페이지의 경로 표시 (`Dashboard > 사용자 관리 > 사용자 목록` 등)
*   **공통 푸터 (Footer)** (선택)
    *   저작권 문구 등 표시

## 2. 권한 및 계정 관리 (IAM)
어드민 시스템의 가장 핵심적인 기능 중 하나입니다.

*   **회원/사용자 관리 (User Management)**
    *   사용자 목록 조회 (페이징, 검색, 필터링)
    *   사용자 상세 정보 조회 및 수정
    *   사용자 추가/삭제 (혹은 비활성화/탈퇴 처리)
*   **역할/권한 관리 (Role-Based Access Control - RBAC)**
    *   관리자, 중간 관리자, 일반 사용자 등 역할 부여
    *   역할에 따른 메뉴 접근 제어 (라우팅 가드 확장)
    *   버튼/기능 단위의 세션/권한 제어

## 3. 공통 컴포넌트 & 유틸리티
어드민 시스템에서 반복적으로 사용되는 UI 패턴을 공통 컴포넌트로 빼두면 생산성이 크게 향상됩니다.

*   **데이터 테이블 (Data Table)**
    *   정렬, 필터링, 검색, 페이지네이션 기본 제공 컴포넌트 (`antd` Table 기반 확장)
    *   일괄 채크박스(Bulk Action) 처리 기능
*   **입력 폼 컴포넌트 (Forms)**
    *   에러 메시지 처리, 유효성 검증(Validation)이 내장된 폼 세트
    *   날짜 선택기(Date Picker), 시간 선택기, 범위 선택 컴포넌트
    *   멀티 셀렉트 및 자동완성(Autocomplete) 검색 인풋
*   **다이얼로그/모달 (Modal) 관리**
    *   확인(Confirm), 알림(Alert), 입력(Prompt) 모달을 쉽게 호출할 수 있는 전역 Store 또는 Context 기반 헬퍼 함수
*   **로딩 및 에러 처리 (Fallback)**
    *   전역 스피너 또는 페이지 단위 스켈레톤(Skeleton) 로딩 UI
    *   API 실패 시 (401, 403, 500 에러) 공통 에러 토스트(Toast) 처리 (현재 `api-client` 확장)

## 4. 콘텐츠 관리 (CMS 핵심)
실제 관리할 데이터를 다루는 부분입니다. 비즈니스 성격에 따라 다르지만 보통 아래와 같은 내용이 들어갑니다.

*   **게시물 관리 / 공지사항 (Board Management)**
    *   공지사항, 자주 묻는 질문(FAQ) 등 기본 게시판 생성/수정/삭제
    *   WYSIWYG 에디터 연동 (예: React Quill, Toast UI Editor 등)
*   **파일 업로드 관리 (File Management)**
    *   이미지 업로드 모듈 (S3 등 클라우드 스토리지 연동)
    *   드래그 앤 드롭 업로드 컴포넌트 및 파일 미리보기 (CDN 연동)

## 5. 시스템 및 인프라
*   **차트 및 대시보드 위젯 (Dashboard Metrics)**
    *   일일 접속자 수, 매출, 주요 지표 요약을 보여주는 그래프 컴포넌트 (Recharts, Chart.js 등 연동)
*   **활동 로그 조회 (Audit Log)**
    *   누가, 언제, 어떤 데이터를 수정했는지 확인하는 보안 로깅 테이블
*   **엑셀 입/출력 모듈 (Excel Export/Import)**
    *   테이블 데이터를 `.xlsx` 혹은 `.csv`로 다운로드 하거나 대량 등록을 위한 업로드 기능

### 💡 제안하는 첫 번째 (Next Step) 개발 로직: **레이아웃(Layout) 개선**
가장 먼저 눈에 보이는 **레이아웃** 요소(Sidebar와 Header)를 추가하는 것을 추천합니다. 
`src/layouts/protected-layout.jsx` 파일을 수정하여, `antd`의 `<Layout>` 컴포넌트를 활용해 왼쪽에 내비게이션 메뉴바를 스캐폴딩하고 우측에 본문 레이아웃을 잡는 기본 작업을 우선적으로 진행해 보시는 것이 좋습니다. 
원하신다면 제가 해당 **`ProtectedLayout`의 기본 템플릿(Sider + Header + Content) 구조 스캐폴딩** 작업을 먼저 도와드릴 수 있습니다.
