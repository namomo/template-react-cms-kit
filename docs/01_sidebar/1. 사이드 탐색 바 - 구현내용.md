# 1. 사이드 탐색 바 (Sidebar) - 구현 내용

## 1. 개요
어드민 사이트의 좌측을 고정적으로 담당할 사이드 네비게이션(LNB)의 구현 내역입니다.

## 2. 작업 내용
### 2-1. 패키지 의존성 추가
*   사이드바 및 헤더에 아이콘을 사용하기 위해 `@ant-design/icons` 패키지를 설치했습니다.

### 2-2. `ProtectedLayout` 구조 개편 (`src/layouts/protected-layout.jsx`)
*   Ant Design의 `<Layout>` 컴포넌트를 활용하여 화면 전체를 덮는 뼈대(`minHeight: '100vh'`)를 구성했습니다.
*   **좌측 사이드바 (`<Sider>`)**
    *   `<Menu>` 컴포넌트에 `theme="dark"`, `mode="inline"`을 적용하여 어드민 패널에 적합한 디자인을 적용했습니다.
    *   `Dashboard`, `사용자 관리`, `설정` 등 임시 메뉴 아이템을 정의하고 랜더링 하였습니다.
    *   `react-router-dom`의 `useLocation`과 연동하여 현재 주소(`location.pathname`)와 일치하는 메뉴가 자동으로 활성화기능(Highlight) 되도록 구성했습니다.
    *   메뉴 클릭 시 `useNavigate`를 사용하여 등록된 `key`(주소) 값으로 라우팅을 이동하도록 처리했습니다.
*   **상단 헤더 (`<Header>`)**
    *   사이드바를 접고 펼칠 수 있는 토글 버튼(`Hamburger icon`)을 배치하여 공간 활용도를 높일 수 있도록 구성 완료했습니다.
    *   우측 영역에 임시로 로그인한 사용자임을 알리는 텍스트 표기 및 기존에 있던 '로그아웃' 기능을 버튼 형태(`useAuthStore`의 `clearAuth` 함수 연동)로 부착했습니다.
*   **본문 영역 (`<Content>`)**
    *   기존의 `<Outlet />`을 `<Content>` 태그로 감싸고, 적절한 여백(margin, padding)과 배경색을 지정하여 실제 페이지 내용이 깔끔하게 보이도록 틀을 마련했습니다.

## 3. 완료 및 다음 작업 (Next Steps)
*   **완료**: 사이드바 레이아웃 구조 세팅 및 접기/펴기 (Toggle) 기능 구현 완료.
*   **보완 예정**: 라우터 경로에 맞는 실제 페이지가 아직 모두 준비되지 않아 하드 코딩 된 부분(예: `/users`, `/settings` 라우트 컴포넌트 추가 필요)은 추후 각 항목 구현 시 확장될 예정입니다.
