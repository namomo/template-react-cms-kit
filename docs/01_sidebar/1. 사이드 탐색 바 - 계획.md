# 1. 사이드 탐색 바 (Sidebar) - 계획

## 1. 개요
어드민 사이트의 좌측을 고정적으로 담당하며, 주요 메뉴간의 이동을 책임질 사이드 네비게이션(LNB: Local Navigation Bar) 기획안입니다.

## 2. 목적
사용자가 대시보드, 사용자 관리 등 CMS 내의 다양한 페이지로 직관적이고 빠르게 이동할 수 있도록 하며, 접기/펴기 기능을 통해 메인 콘텐츠 영역을 넓게 사용할 수 있도록 지원합니다.

## 3. 구현 내용 및 요구 사항
### 3-1. 레이아웃
*   **위치**: `ProtectedLayout` 내부 좌측 (화면 좌측 전체 높이 차지, 고정 레이아웃)
*   **컴포넌트**: Ant Design(antd)의 `Layout.Sider` 및 `Menu` 컴포넌트를 활용합니다.

### 3-2. 요구 기능
1.  **메뉴 아이템 렌더링**: 라우트 정보에 맞춰 1Depth, 2Depth 형태의 메뉴 리스트를 화면에 표시합니다.
2.  **현재 경로 연동**: 현재 URL 경로(pathname)에 맞춰 선택된 메뉴(Active state)가 시각적으로 표시되어야 합니다.
3.  **라우팅 이동**: 메뉴 아이템 클릭 시, `react-router-dom`의 `useNavigate` 훅을 사용해 해당 페이지로 클라이언트 사이드 라우팅이 처리되어야 합니다.
4.  **접기/펴기 (Collapse)**: 사이드바의 너비를 좁은 형태(아이콘만 표시) 혹은 넓은 형태(아이콘+텍스트 표시)로 토글할 수 있는 하단 버튼 혹은 토글 스위치를 제공해야 합니다.

### 3-3. 대상 파일
*   **수정 예정**:
    *   `src/layouts/protected-layout.jsx`: Ant Design `Layout` 기반 구조 적용. 사이드바 렌더링 추가.
*   **추가 예정**:
    *   (선택) 메뉴 데이터를 중앙화하기 위한 설정 파일(e.g., `src/constants/menu.js` 등) 혹은 라우트 기반 자동 생성 방식 적용 검토.

## 4. 진행 순서 (Next Steps)
1. `ProtectedLayout` 컴포넌트 리팩토링 (antd Layout 적용).
2. 하드코딩된 임시 메뉴(Dashboard, Settings 등)로 사이드바 렌더링 및 스타일 확인.
3. `useNavigate` 와 라우팅 연결 테스트.
4. 메뉴 클릭 및 접기/펴기 작동 확인.
