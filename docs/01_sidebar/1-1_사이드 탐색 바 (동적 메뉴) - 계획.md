# 1-1. 사이드 탐색 바 (동적 메뉴) - 계획

## 1. 개요
현재 사이드바(`ProtectedLayout`)에 하드코딩 되어 있는 메뉴 리스트를 개선하여, 서버(API)로부터 메뉴 구성(권한별 메뉴 등)을 동적으로 받아와 렌더링하도록 변경합니다. API 호출 실패 또는 응답이 없을 경우에는 로컬에 정의된 기본(Fallback) 메뉴 파일에서 데이터를 읽어와 표시하는 안전장치(Fail-safe)를 마련합니다.

## 2. 목적
*   **유연성 확보**: 사용자 권한(Admin, 일반 사용자 등)이나 백엔드의 메뉴 관리 설정에 따라 동적으로 메뉴 구조를 변경할 수 있습니다.
*   **안정성 보장**: 백엔드 서버에 일시적인 문제가 발생하거나 메뉴 구성 데이터가 비어있을 경우, 사이트 탐색이 불가능해지는 치명적인 에러를 막기 위해 로컬 설정 파일을 사용합니다.

## 3. 구현 내용 및 요구 사항
### 3-1. 메뉴 데이터 구조
서버 및 로컬이 반환/저장하는 메뉴 데이터는 다음과 같은 JSON 형태의 배열 구조를 가집니다.
```json
[
  { "key": "/dashboard", "icon": "DashboardOutlined", "label": "Dashboard" },
  { "key": "/users", "icon": "UserOutlined", "label": "사용자 관리" }
]
```
*(참고: Ant Design 아이콘은 순수 문자열 데이터를 기반으로 동적 렌더링되도록 맵퍼(Mapper) 함수 처리가 필요합니다.)*

### 3-2. 로컬 Fallback (기본 메뉴) 분리
*   **파일**: `src/constants/default-menu.js`
*   서버 통신 실패 시 사용할 기본 배열 상수를 정의합니다.

### 3-3. 서비스 모듈 생성 (API 통신)
*   **파일**: `src/services/menu.service.js`
*   `getMenuItems()` 함수를 구현하여, `api-client.js`를 통해 `/menu` API를 호출합니다.
*   `try-catch` 블록으로 예외를 잡아채고, 에러 발생 시 `default-menu.js`의 객체를 반환하도록 구현합니다.

### 3-4. 컴포넌트 연동 (상태 관리)
*   **파일**: `src/layouts/protected-layout.jsx`
*   컴포넌트 마운트 시(`useEffect`) `getMenuItems()`를 호출하여 응답받은 메뉴 데이터를 `useState`에 저장합니다.
*   응답받은 아이콘 문자열("UserOutlined")을 실제 Ant Design 아이콘 컴포넌트(`<UserOutlined />`)로 변환해주는 유틸리티 로직을 추가하여 `<Menu>`의 `items` prop으로 전달합니다.

## 4. 진행 순서
1. `src/constants/default-menu.js` 파일에 풀백 메뉴 정의.
2. `src/services/menu.service.js` 파일에 API fetch 및 풀백 처리 로직 작성.
3. `src/layouts/protected-layout.jsx`에서 메뉴 상태 도입 및 아이콘 동적 렌더링 매핑 구현.
4. (계획 완료 후) 작업 완료 문서 작성 및 전체 작업 목록 갱신.
