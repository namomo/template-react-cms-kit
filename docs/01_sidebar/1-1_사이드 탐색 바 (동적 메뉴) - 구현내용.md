# 1-1. 사이드 탐색 바 (동적 메뉴) - 구현 내용

## 1. 개요
사이드바의 메뉴 항목을 프론트엔드 코드에 직접 하드코딩하지 않고, 서버(API) 응답 또는 로컬 기본(Fallback) 파일에서 불러오도록 개선한 내역입니다.

## 2. 작업 내용
### 2-1. 기본 메뉴(Fallback) 데이터 분리
*   **생성 파일**: `src/constants/default-menu.js`
*   기존 `ProtectedLayout` 내부에 배열로 존재하던 `menuItems` 상수들을 독립된 파일로 분리했습니다.
*   서버 에러 시 등에 대안으로 사용할 수 있는 안전장치 역할을 합니다.
*   *참고: 아이콘은 더 이상 컴포넌트 객체가 아닌, 서버 전송에 적합한 순수 문자열 형태(`"UserOutlined"`)로 저장됩니다.*

### 2-2. 메뉴 전용 API 서비스 생성
*   **생성 파일**: `src/services/menu.service.js`
*   `api-client.js`를 사용해 `/menu` API를 호출하는 `getMenuItems` 비동기 함수를 작성했습니다.
*   현재는 테스트 서버가 없으므로 API 호출부를 주석 처리하고, `setTimeout`을 통해 0.3초간 로딩을 지연 시킨 후 `default-menu.js` 데이터를 반환하도록 Mock(가짜) 함수로 구현했습니다. (실제 요청 실패 또는 지연을 가정한 안전한 대응 로직)

### 2-3. `ProtectedLayout` 동적 로딩 적용 (`src/layouts/protected-layout.jsx`)
*   **메뉴 상태 도입 및 로딩 처리**:
    *   `menuItems`와 `loading` 상태(`useState`)를 추가했습니다.
    *   초기 렌더링 시 `<Spin>` 컴포넌트를 이용해 로딩 인디케이터가 표시되도록 했습니다.
*   **`useEffect`를 통한 데이터 패칭**:
    *   컴포넌트가 마운트되고, 사용자가 인증된 상태(`isAuthenticated === true`)일 경우 `getMenuItems()`를 호출합니다.
*   **동적 아이콘 매핑 (Icon Mapper)**:
    *   서버나 로컬 모듈에서 전달받은 아이콘 이름(string)을 안트 디자인의 실제 리액트 아이콘 컴포넌트로 변환해주기 위한 `iconMapper` 딕셔너리 객체를 생성했습니다. 이를 통해 안전한 동적 컴포넌트 생성이 가능해졌습니다.
