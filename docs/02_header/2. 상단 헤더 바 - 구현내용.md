# 02. 상단 헤더 바 (Header) - 구현 내용

## 1. 개요
기존에 `ProtectedLayout` 내부에 임시로 렌더링되던 상단 헤더(Header) 영역을 개선하여, 실제 서비스에 필요한 공통 기능(테마 변경, 사용자 프로필, 로그아웃)을 모듈화하여 구현한 내역입니다.

## 2. 작업 내용
### 2-1. 테마 상태 관리 (다크모드 지원)
* **생성 파일**: `src/stores/theme-store.js`
* `Zustand`를 활용하여 전역 테마 상태(`themeMode`)를 관리하는 스토어를 구성했습니다.
* 사용자가 지정한 테마가 유지될 수 있도록 `localStorage`와 연동하여 상태를 저장/호출(`app-theme` 키 사용)하도록 구현했습니다.
* **적용**: 최상위 통합 파일인 `src/App.jsx`에서 `antd`의 `ConfigProvider` 컴포넌트를 사용해 이 상태값에 따라 앱 전체 알고리즘(`theme.darkAlgorithm`, `theme.defaultAlgorithm`)이 동적으로 전환되도록 적용했습니다.

### 2-2. 테마 토글 버튼 분리
* **생성 파일**: `src/components/layout/theme-toggle.jsx`
* 해(BulbOutlined)/달(BulbFilled) 아이콘을 적용한 스위치(`<Switch>`)를 생성하고 `theme-store`의 `toggleTheme` 액션을 연결했습니다.

### 2-3. 프로필 메뉴 및 로그아웃 기능 분리
* **생성 파일**: `src/components/layout/header-actions-dropdown.jsx`
* 전역 상태 `auth-store`로부터 로그인한 사용자의 정보를 가져와 렌더링합니다.
* 우측 상단의 아바타(`<Avatar>`) 클릭 시 이름과 이메일 정보가 보이고, `로그아웃` 버튼이 포함된 드롭다운(`<Dropdown>`) 메뉴가 펼쳐지도록 구현했습니다.

### 2-4. 레이아웃 CSS 버그 수정
* **수정 파일**: `src/index.css`, `src/App.css`
* Vite 템플릿의 기본 스타일(특히 `body { display: flex }` 와 `#root` 의 `max-width` 등)이 Ant Design 컴포넌트의 가로 확장(전체 너비 100%)을 막고 중앙으로 묶는 문제를 발견하여 레이아웃을 해치는 관련 CSS 로직을 모두 청소했습니다.

## 3. 완료 내역
* **기능 완료**: 상태 기반의 전역 테마(다크모드/라이트모드) 토글, 프로필 확인 및 로그아웃이 가능한 드롭다운 메뉴 완성.
* 헤더 구조를 파일로 쪼개어 `ProtectedLayout` 컴포넌트 내부 코드를 한층 간결하게 정리했습니다.
