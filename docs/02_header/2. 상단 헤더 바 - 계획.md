# 02. 상단 헤더 바 (Header) - 계획

## 1. 개요
현재 임시로 렌더링되고 있는 상단 영역을 개선하여, 실제 어드민/CMS 환경에 필요한 공통 GNB(Global Navigation Bar) 항목들을 구현합니다. `ProtectedLayout` 내의 Ant Design `<Header>` 컴포넌트를 확장하여 작업합니다.

## 2. 목적
*   **사용자 인지 및 편의성 강화**: 현재 로그인한 사용자의 정보를 명확히 표시하고, 다크모드/라이트모드 등 시스템 테마 설정을 손쉽게 변경할 수 있는 접근성을 제공합니다.
*   **기본 기능 제공**: 로그아웃 등 필수적인 계정 관리 기능을 헤더에 배치하여 일관된 사용자 경험을 유지합니다.

## 3. 구현 내용 및 요구 사항
### 3-1. 레이아웃 구조 (`src/layouts/protected-layout.jsx` 수정)
*   **좌우 분할 렌더링**: Flexbox를 활용해 좌측에는 접기/펴기 버튼이나 로고 등을 배치하고, 우측 끝에는 사용자 관련 메뉴 컴포넌트들을 배치합니다.
*   **컴포넌트 분리**: 헤더 내부 로직이 복잡해지는 것을 방지하기 위해 우측 메뉴 영역을 별도의 컴포넌트로 분리합니다.

### 3-2. 테마 토글 (Dark/Light Mode) 컴포넌트 생성
*   **버튼 UI**: 태양/달 형태의 토글 스위치 혹은 아이콘 버튼 사용.
*   **상태 관리**: Zustand 등을 활용하거나, 최소한 `ProtectedLayout` 레벨에서 테마 상태를 관리하고 `antd`의 `ConfigProvider`에 전달하여 전체 사이트 테마를 즉시 전환할 수 있도록 구현해야 합니다. (선행 작업 필요 여부 파악 必)

### 3-3. 사용자 프로필 액션 콤보 생성
*   **Dropdown 메뉴 액션**: 아바타(Avatar) 또는 사용자 이름을 클릭했을 때 아래로 펼쳐지는 `Dropdown` 메뉴를 생성합니다.
*   **아이템 구성**:
    *   `로그아웃`: 클릭 시 전역 인증 상태(`useAuthStore`)를 초기화(`logout()`)하고 `/login` 페이지로 리다이렉트 처리.
*   **데이터 연동**: Zustand 스토어(`useAuthStore`)에서 가져온 사용자 이름, 이메일 등의 프로필 텍스트 데이터를 아바타 우측에 노출합니다.

## 4. 진행 순서
1. `src/hooks/use-theme.js` (또는 전역 테마 스토어) 생성 및 설정 (테마 토글 로직 구현용).
2. `src/components/layout/` 내부에 `header-actions-dropdown.jsx` (로그아웃 등), `theme-toggle.jsx` 파일 분리 생성.
3. `src/layouts/protected-layout.jsx` 의 `<Header>` 영역 부분을 편집하여 위 컴포넌트들을 삽입.
4. 다크모드/라아트모드 동작 확인 및 로그아웃 기능 정상 작동 테스트.
